## 一、服务器启动命令
python -m sglang.launch_server \
  --model-path deepseek-ai/DeepSeek-R1-Distill-Qwen-32B \
  --speculative-algorithm SUFFIX \
  
  # 核心参数优化（解决重复生成）
  --speculative-num-draft-tokens 3  `# 大幅减少投机令牌数，降低重复风险` \
  --speculative-suffix-max-tree-depth 16  `# 减小后缀树深度，减少重复模式捕获` \
  --speculative-suffix-max-spec-factor 0.5  `# 严格限制投机数量，提高投机质量` \
  --speculative-suffix-min-token-prob 0.2  `# 提高最小概率阈值，过滤低质量重复令牌` \

  --speculative-accept-threshold-single 0.6  `# 提高单个令牌接受阈值` \
  --speculative-accept-threshold-acc 0.7  `# 提高累积接受阈值` \
  
  # 其他推荐参数
  --trust-remote-code \
  --tp-size 8 \
  --dp-size 1 \
  --host 0.0.0.0 \

## 三、代码修改说明

### 1. 验证阶段应用完整重复惩罚
**文件**: `python/sglang/srt/speculative/ngram_info.py`
**修改内容**: 将简化的线性惩罚替换为完整的重复惩罚应用
**效果**: 在验证阶段对每个投机令牌位置应用完整的重复惩罚机制

### 2. 投机生成阶段添加重复过滤
**文件**: `python/sglang/srt/speculative/suffix_cache_adapter.py`
**修改内容**:
- 在投机令牌生成后添加重复序列过滤步骤
- 实现 `_filter_repetitive_sequences` 方法，包含三层重复检测：
  1. 连续重复令牌检测（最多2个连续重复）
  2. 近期上下文重复检测（10个令牌窗口内限制）
  3. 重复模式检测（如A-B-A-B模式）
**效果**: 从源头上减少投机生成的重复令牌

## 四、预期效果

1. **CEVAL评分提升**: 预计从0.77提升到0.82-0.84，接近理论值0.85
2. **重复生成减少**: 连续重复令牌从平均5-10个减少到2个以内
3. **生成质量提升**: 推理过程更加连贯，减少无意义的重复内容
4. **性能影响**: 推理速度可能下降15-20%，但准确性显著提升
